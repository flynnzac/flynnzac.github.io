Non-Data Projects

I write programs sometimes for non-data things. Usually either to solve a a non-work problem or just for fun.

On the bus to work in DC (circa 2018), I started writing a programming language that I thought would work well for quickly spinning up interactive programs... and would make the busride less boring. It has gotten seriously out of hand and now has a prototype-based object system, multi-tasking, etc. Practically, it's a quick way to spin up a small program that needs to be interactive. I've used it to write several programs to organize myself. It's also just fun to write.

See: the manual and the Github: https://github.com/flynnzac/slobil.

I wrote the “Little Extensible Accounting System” or Leas when I was budgeting for my wedding. It’s kind of the Emacs of personal finance (without the popularity). You can extend it by writing Scheme (a Lisp dialect). Most of the system is written in Scheme that calls C functions. I use the extensibility to do things like fetch the latest stock/mutual fund prices, etc.

See: https://github.com/flynnzac/leas.

pushmac is a small macro language that aims to be a flexible way to template code and other text. I used it for a decent part of SLOBIL.

See: https://github.com/flynnzac/pushmac.
